    Typora+PicGoo 配置 gitee 图床

    Brod-Roy 12/12/2022 MarkDownTypoarPicGo

    # Typora+PicGoo 配置 gitee 图床详讲

    # 一、前言

    相信大部分程序员应该都习惯使用markdown格式来记录笔记。我个人偏好使用Typora来编写笔记，写完后使用CSDN自带的导入功能直接导入也很方便。但是markdown的本地图片导入CSDN后还需要重新上传，很麻烦。

    于是就想着Typora+PicGoo配置gitee图床。但有点需要注意的是**部分朋友反馈gitee会存在图片崩溃的问题（某段时间）**所以有条件，且对稳定性要求比较高的朋友还是建议搭建一个自己的服务器存储比较好。

    # 二、准备工作以及流程

    # 1、操作流程简介

    首先我们需要先准备一个gitee账号，来创建我们的仓库；随后我们需要配置好PicGo对仓库上传图片，验证是否配置成功；配置成功后对接Typora，验证是否可以通过Typora上传图片，验证通过则是成功了。

    # 2、工具及下载地址

    以下是需要准备的工具以及相关网址与下载地址：

    • Typora下载地址：Typora 1.4.8 – 2022最新Typora破解激活教程 (opens new window)

      不想用百度云的可以来 gitee 克隆:MarkDown · Brod_Roy/MarkDown (opens new window)

    • gitee 地址：Gitee.com (opens new window)

    • PicGo 下载地址：Releases · Molunerfinn/PicGo (opens new window)

    # 三、搭建 gitee 仓库

    gitee 地址：Gitee.com (opens new window)

    # 1、创建账号

    如果你没有注册过，就需要先注册一个账号，有账号的登录就好了

    # 2、新建仓库

    1. 点击主页的右上角的 + 号，打开新建仓库

    1. 填写仓库的相关信息，需要记住的就是你的仓库名称，稍后需要在 PicGo 上填写你的仓库地址和 gitee 账户名称。要注意的是图中的我默认是私有项目，我们需要在仓库创建成功后将私有改成开源，如果你默认是开源就不用了。此外必须勾选初始化仓库和设置模板→Readme文件

    2. 完成后会来到仓库页面，因为我们是私有仓库，需要先把私有变开源仓库，如果你是默认开源则可以跳过这个步骤。点击管理跳转仓库管理页面，把这个开源选项勾上即可。设置开源完成后，我们的仓库创建步骤就完成啦。

    # 3、设置 gitee 令牌

    我们需要设置一个私人令牌，来帮助PicGo在上传图片时进行验证，我们一定要保存好我们的令牌，不能暴露给别人，也注意不要丢失，否则我们需要重新进行配置。

    1. 鼠标移动到我们的头像，找到设置的按钮，我们需要在设置页面生成令牌

    2. 点击左侧栏的私人令牌，可以看到我已经生成了一些令牌了，我们接下来点击生成令牌，来创建一个新的令牌

    3. 我们生成令牌只需要勾选上面两个即可，其他的权限我们就不需要啦，勾选后为你的令牌命名，至少知道它是用来干什么的，点击提交进行验证。

    4. 验证成功后就会显示你的私人令牌了，你需要找个地方把他储存下来，比如你的Typora或者记事本什么的地方，我们在稍后配置PicGo时候再使用。如果不小心关闭了，删掉重新配置就好，无伤大雅。

    # 四、配置 PicGo 上传图片

    我们需要在 PicGo 中验证是否可以完成图片的上传，我在刚刚配置的时候在这里卡了很久，我会把我遇到的一些常见的问题列出来，你们可以看看有没有出现同样的问题。

    # 1、下载 PicGo

    根据 PicGo下载地址 (opens new window) 下载对应的版本：

    # 2、安装 gitee 插件

    点击 PicGo 的侧边栏 【插件设置】 搜索 gitee

    # 3、配置 gitee 设置

    重点就是第一行 repo：gitee 账户名称/仓库名称，还有 token：生成的gitee令牌token，这两个地方

    # 4、修改上传选项

    点击上传区，把默认的上传选项改成gitee，我就是因为没有选对这个，导致折腾了好久都没配置成功

    # 5、测试上传图片

    点击上传图片，上传成功后右下角出现提示即可，到这一步 PicGo 的配置就完成了

    # 五、配置 Typora 接入 PicGo

    配置好 PicGo 后需要在 Typora 接入，完成最后的上传服务。

    # 1、打开选项

    打开任意的 Typora 文件，选择文件左上角的【文件】→【偏好设置】

    # 2、配置 Typora 设置

    上传服务选择 PicGo ，接着去配置到 PicGo.exe 对应的文件路径即可

    # 3、验证上传图片成功

    最后一步就是点击验证图片上传选项，看看是否可以上传成功，如果提示成功即配置完成，可以在 Typora 里一键上传图片啦。

    # 六、Typora 使用上传服务

    当我们复制图片粘贴到 Typora 时，就会弹出上传图片的提示，点击即可一键上传图片，同时替换你本地的图片路径为网络地址。

    到了这一步我们的操作就全部完成了，如果有帮助到你，希望可以给我一个赞👍哦。
